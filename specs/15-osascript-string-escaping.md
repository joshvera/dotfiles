# Spec: Osascript String Escaping

## Purpose
Fix silent notification failures caused by unescaped quotes and backslashes in notification messages. Malformed osascript strings fail to send notifications without error messages, leaving users without feedback.

## Jobs to Be Done

### Job 1: Handle Quotes in Summaries
When Haiku summary contains quotes:
- Summary: `The user asked "what is the meaning?"`
- Current outcome: osascript command syntax error, notification fails silently
- Expected outcome: Quotes properly escaped, notification displays

### Job 2: Handle Backslashes in Summaries
When summary contains backslashes (file paths, regex, etc.):
- Summary: `Processing file: C:\Users\name\file.txt`
- Current outcome: osascript interprets backslash as escape, command fails silently
- Expected outcome: Backslashes properly escaped, notification displays

### Job 3: Handle Newlines in Summaries
When summary from transcript contains newlines:
- Summary spans multiple lines in truncated response
- Current outcome: osascript string breaks across lines, command fails
- Expected outcome: Newlines escaped/removed, notification displays

### Job 4: No Silent Failures
Currently, osascript errors are silently suppressed:
```bash
osascript -e "display notification..." 2>/dev/null || true
```
User doesn't know notification failed to send.
Expected: Failures logged, user can troubleshoot.

## Inputs
- Title (always simple: "Claude Code: projectname")
- Message/summary (contains user content, URLs, quotes, etc.)
- Event ID (safe, generated)

## Outputs
- Properly escaped osascript command
- Notification successfully displays (or error logged)
- No silent failures

## Problem Statement

### Current Implementation (BROKEN)
```bash
osascript -e "display notification \"$message\" with title \"$title\"" 2>/dev/null || true
```

**Issues:**
1. Quotes in `$message` break the command:
   - Input: `The user said "hello"`
   - Command: `display notification "The user said "hello"" with title...`
   - Result: Syntax error, silent failure

2. Backslashes in `$message` are interpreted by osascript:
   - Input: `Processing C:\Users\name\file.txt`
   - osascript sees `\U` and `\n` as escape sequences
   - Command: `display notification "Processing C:Users ame..."`
   - Result: Wrong text or syntax error

3. Newlines break the command:
   - Input: Summary with embedded newline
   - Command spans multiple lines
   - Result: Syntax error

4. Errors are suppressed with `2>/dev/null || true`:
   - osascript returns non-zero on syntax error
   - Error output is discarded
   - User has no way to know notification failed

### Example Failure Case
```bash
# Summary generated by Haiku API
summary='Claude refactored the "build" command with better error handling.\nCheck the logs for details.'

# Current code
osascript -e "display notification \"$summary\" with title \"Claude Code: myproject\"" 2>/dev/null || true

# osascript sees:
# display notification "Claude refactored the "build" command with better error handling.
# Check the logs for details." with title "Claude Code: myproject"
#
# Result: Syntax error (mismatched quotes, newline in string)
# Notification fails silently
```

## Escaping Strategy

### Approach: Proper osascript String Escaping
OSA Script (AppleScript) requires:
1. Double quotes escaped as `\"`
2. Backslashes escaped as `\\`
3. Newlines converted to spaces or `\n` within string

The shell (bash) ALSO does escaping before passing to osascript:
- Need to escape for both bash AND osascript
- Use bash's `${variable//pattern/replacement}` syntax

**Chain of escaping:**
1. User input contains: `He said "hi"`
2. Bash processes: `"He said \"hi\""` (escapes one level)
3. osascript receives: `He said "hi"` (bash interpreter unwraps)
4. AppleScript sees: `He said "hi"` in string literal (correct)

### Better Approach: Use jq for Escaping
jq handles all escaping correctly for us:
```bash
message=$(jq -n --arg msg "$message" '$msg')
```
Then build osascript command with jq-escaped string.

**Even better:** Use jq with `-Rs` flag for raw string processing:
```bash
jq -n --arg msg "$message" -r '@text "\($msg)"'
```

### Recommended: bash Native Escaping + Validation
Use bash parameter expansion for escaping:
```bash
# Escape double quotes
local escaped_msg="${message//\"/\\\"}"

# Escape backslashes (must do this first!)
escaped_msg="${escaped_msg//\\/\\\\}"

# Remove newlines
escaped_msg="${escaped_msg//$'\n'/ }"

# Build command with escaped message
osascript -e "display notification \"$escaped_msg\" with title \"$title\""
```

## Dependencies
- osascript (built-in macOS)
- bash (already required for script)
- No new external dependencies

## Key Decisions

### Decision 1: Use bash Parameter Expansion for Escaping
Avoid shell injection risks by using bash's native `${var//pattern/replacement}` syntax instead of sed/awk.

### Decision 2: Escape Backslash FIRST
Order matters! Must escape `\` before `"`:
```bash
# CORRECT ORDER
escaped="${msg//\\/\\\\}"  # Escape backslash first
escaped="${escaped//\"/\\\"}"  # Then escape quotes
```
Reverse order causes double-escaping.

### Decision 3: Convert Newlines to Spaces
Remove embedded newlines rather than trying to escape them in osascript. Notification UI doesn't display multi-line text anyway.

### Decision 4: Log Errors with Full Context
Instead of `2>/dev/null || true`, capture errors:
```bash
if ! output=$(osascript -e "..." 2>&1); then
    echo "$(date): osascript failed: $output" >> /tmp/claude-hook-debug.log
fi
```

### Decision 5: Keep Function Simple
Create new internal function `_escape_for_osascript()` to handle escaping, then reuse in both notification functions.

## Implementation

### Helper Function: Escape for osascript
```bash
# Escape string for use in osascript command
# Handles quotes, backslashes, newlines
_escape_for_osascript() {
    local input="$1"
    local output="$input"

    # Escape backslashes FIRST (must be before quotes)
    output="${output//\\/\\\\}"

    # Escape double quotes
    output="${output//\"/\\\"}"

    # Remove newlines (convert to space)
    output="${output//$'\n'/ }"

    echo "$output"
}
```

### Updated: send_desktop_notification_with_click_handler()
```bash
send_desktop_notification_with_click_handler() {
    local title="$1"
    local message="$2"
    local event_id="$3"

    if [[ -z "$title" || -z "$message" || -z "$event_id" ]]; then
        echo "$(date): send_desktop_notification_with_click_handler: missing required parameters" >> /tmp/claude-hook-debug.log
        return 1
    fi

    # Get state directory for cancel marker path
    local state_dir
    state_dir=$(initialize_state_dir) || return 1

    local cancel_marker_path="${state_dir}/.cancel-${event_id}"

    # Escape message for osascript
    local escaped_message
    escaped_message=$(_escape_for_osascript "$message")

    local escaped_title
    escaped_title=$(_escape_for_osascript "$title")

    # Try terminal-notifier with click handler
    if command -v terminal-notifier >/dev/null 2>&1; then
        # terminal-notifier execute handler: create cancel marker on click
        local output
        if output=$(terminal-notifier \
            -title "$escaped_title" \
            -message "$escaped_message" \
            -execute "touch '$cancel_marker_path'" \
            -sender "com.ghostty.terminal" \
            2>&1); then
            echo "$(date): Desktop notification sent via terminal-notifier for event: $event_id" >> /tmp/claude-hook-debug.log
        else
            echo "$(date): terminal-notifier failed: $output. Falling back to osascript" >> /tmp/claude-hook-debug.log
            # Fallback to osascript
            if ! output=$(osascript -e "display notification \"$escaped_message\" with title \"$escaped_title\"" 2>&1); then
                echo "$(date): osascript notification failed: $output (event: $event_id)" >> /tmp/claude-hook-debug.log
                return 1
            fi
        fi
    else
        # Fallback: osascript without click handler
        if ! output=$(osascript -e "display notification \"$escaped_message\" with title \"$escaped_title\"" 2>&1); then
            echo "$(date): osascript notification failed: $output (event: $event_id)" >> /tmp/claude-hook-debug.log
            return 1
        fi
        echo "$(date): Desktop notification sent via osascript for event: $event_id (terminal-notifier not available)" >> /tmp/claude-hook-debug.log
    fi

    # Mark notification sent in metadata
    update_event_field "${event_id}" "desktop_notified" "true"

    echo "$(date): Desktop notification sent for event: $event_id (click handler enabled if terminal-notifier available)" >> /tmp/claude-hook-debug.log
}
```

### Updated: send_idle_notification() [ntfy variant]
ntfy uses curl, different escaping rules. Ensure curl data is properly quoted:
```bash
send_idle_notification() {
    local summary="${1:-}"

    # Use env vars consistent with zsh notify function
    local ntfy_url="${NTFY_URL:-}"
    local ntfy_topic="${NTFY_TOPIC:-wintermute}"

    if [[ -z "$ntfy_url" ]]; then
        return 0  # No ntfy configured
    fi

    # ... (title, session_info construction) ...

    local message
    if [[ -n "$summary" ]]; then
        # curl -d handles newlines and quotes differently than osascript
        # Just ensure proper JSON if applicable, or plain text
        message="${summary}${session_info}"
    else
        message="Waiting for input (idle >${IDLE_TIMEOUT}s)${session_info}"
    fi

    # curl -d sends as-is, no escaping needed for plain text
    curl -s --max-time 5 \
        -H "Title: $title" \
        -H "Tags: $tags" \
        -d "$message" \
        "$ntfy_url/$ntfy_topic" > /dev/null 2>&1 || {
        echo "$(date): ntfy notification failed" >> /tmp/claude-hook-debug.log
    }
}
```

## Edge Cases

| Case | Handling |
|------|----------|
| Empty message | osascript sends notification with empty body (acceptable) |
| Message > 2000 chars | osascript truncates display, no command failure |
| Unicode characters (emoji, Chinese, etc.) | bash/osascript handle UTF-8 natively |
| Quotes and backslashes together | Both escaped correctly via ordering |
| Tab characters | Preserved (not special to osascript) |
| Single quotes in message | No escaping needed (not special in double-quoted string) |
| Very long title | osascript truncates in UI, no command failure |
